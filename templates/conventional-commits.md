# Conventional Commits — RU-шпаргалка

Английский префикс + русское описание. Глагол в инфинитиве. Без точки. ≤72 символа в заголовке.

## Префиксы

| Префикс | Когда использовать | Пример |
|---|---|---|
| `feat:` | Новая функциональность для пользователя | `feat: добавить модуль стейкинга` |
| `fix:` | Исправление бага | `fix: исправить расчёт комиссии в swap` |
| `docs:` | Только документация | `docs: обновить README — раздел Quick Start` |
| `refactor:` | Рефакторинг без изменения поведения | `refactor: вынести логику авторизации в отдельный сервис` |
| `test:` | Добавление/изменение тестов | `test: добавить тесты для модуля платежей` |
| `chore:` | Зависимости, конфиги, рутина | `chore: обновить зависимости` |
| `ci:` | Изменения CI/CD | `ci: настроить сборку через GitHub Actions` |
| `style:` | Форматирование, без изменения логики | `style: отформатировать код через prettier` |
| `perf:` | Оптимизация производительности | `perf: оптимизировать запросы к БД` |
| `build:` | Изменения системы сборки | `build: перейти на Turbopack` |
| `revert:` | Откат предыдущего коммита | `revert: откатить feat: добавить X` |

## Scope (опционально)

Если хочется уточнить область:
```
feat(auth): добавить refresh-токены
fix(swap): исправить округление
docs(api): описать /v1/users
```

## Тело коммита

Через пустую строку. Объясняет **зачем**, а не **что** (что — видно из диффа).

```
feat(auth): добавить refresh-токены

JWT с TTL 15 мин снижает риск компрометации при утечке access-токена.
Refresh-токен живёт 30 дней, хранится в HttpOnly cookie + serverside таблице
revoked_tokens — поддерживает logout-from-all-devices.

Closes #142
```

## Breaking changes

Через `!` после префикса или `BREAKING CHANGE:` в теле:

```
feat!: переход на API v2

BREAKING CHANGE: эндпоинты /api/* переехали в /api/v2/*. Старые URL
возвращают 410 Gone до 2026-09-01, потом удаляются.
```

## Чего не делать

Плохо:
```
fix
update
aaa
final
wip2
fix bug
update README
```

Хорошо:
```
fix: исправить таймаут на медленных подключениях
chore: обновить eslint до v9
docs: добавить пример запроса в README
```

## Авто-changelog

Conventional Commits парсятся стандартными тулзами (`standard-version`, `release-please`, `git-cliff`) → автоматический CHANGELOG.md и semver-теги. Это и есть основная причина соблюдать формат.
