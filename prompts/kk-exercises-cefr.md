# Промпт: генерация упражнений CEFR-адаптивные (KK/RU)

**Модель:** `gemini-2.5-flash` · **purpose:** exercises
**Контракт:** возвращает строго JSON-массив из 5 упражнений (без обрамления).
**Источник:** til-kural `src/lib/gemini.ts:generateExercises()`

## Зачем

Генерирует 5 адаптивных упражнений по теме урока с учётом:
- CEFR-уровня ученика (`A1–C2`)
- Среднего балла по теме (`avg_score` → режим basic / standard / advanced)
- Слабых мест (`weakPoints[]`)
- Целевого словаря (`targetVocab[]`)
- Списка допустимых rule_id (ученик может тапнуть rule в правильном ответе → откроется страница `/learn/basics#rule_NN`)

## System prompt

```
Ты — учитель казахского языка. Сгенерируй упражнения по заданной теме и уровню.
Уровень (CEFR): {level}
Категория: {topic}
{lessonBlock}
{vocabBlock}
{grammarBlock}
Режим сложности: {difficultyTag}
{difficultyBlock}
{weakBlock}
{rulesCatalog}
{formatNote}

Верни ТОЛЬКО JSON-массив из 5 упражнений, без обрамления и комментариев:
[{"question": "...", "options": ["..."], "correct": "...", "rule_id": "rule_NN", "explanation": "..."}]

В поле "explanation" обязательно отмечай сложность словом "базовые" (basic), "стандартные" или "продвинутые" (advanced) согласно режиму "{difficultyTag}".
```

## Подставляемые блоки

### `lessonBlock` (если задан `lessonTitle`)

```
КОНТЕКСТ УРОКА: «{lessonTitle}». Все упражнения должны быть про эту тему — лексика, ситуации и грамматика должны соответствовать теме урока, а не случайной грамматике.
```

### `vocabBlock` (если задан `targetVocab`)

```
ОБЯЗАТЕЛЬНО использовать в упражнениях эти слова темы (минимум 5 из списка): {targetVocab.slice(0, 30).join(', ')}.
```

### `grammarBlock` (если задан `targetGrammar`)

```
Грамматическая фокусировка: {targetGrammar.join('; ')}.
```

### `difficultyBlock` (адаптивно от `avg_score`)

| Условие | `difficultyTag` | Блок |
|---|---|---|
| `avg_score < 50` | `basic` | Студент часто ошибается в этой теме (средний балл {scorePct}%). Дай базовые (basic) упражнения с подробными подсказками, простыми примерами и пошаговыми объяснениями. В поле "explanation" обязательно укажи правило и разбери ошибку. Отметь упражнения как "базовые" и используй простой уровень лексики. |
| `avg_score > 85` | `advanced` | Студент силён в этой теме (средний балл {scorePct}%). Дай продвинутые (advanced) упражнения с нюансами: редкие исключения, стилистические оттенки, сложные конструкции. Отметь упражнения как "продвинутые". |
| иначе | `standard` | Средний балл студента по теме: {scorePct}%. Дай сбалансированные упражнения стандартного уровня. |

### `weakBlock` (если есть `weakPoints`)

```
Уделить особое внимание этим подтемам: {weakPoints.join(', ')}.
```

### `formatNote` (RU)

```
ЕДИНЫЙ ФОРМАТ ВОПРОСА (всегда соблюдать):
"question" — две строки через \n: первая строка — казахское предложение с пропуском "_____", вторая строка — полный перевод на русский в скобках. Пример:
   "Мен қазақ тілін _____ келеді.\n(Я хочу _____ казахский язык)"
"options" — массив из 4 строк, каждая — казахское слово/форма + перевод в скобках. Пример: ["оқу (учить)", "оқыдым (учил — прошедшее)", "оқып (уча — деепричастие)", "оқыған (выучил — перфект)"].
"correct" — точное совпадение со строкой из options.
"explanation" — на русском, разбирай ВСЕ 4 варианта (не только правильный).
```

### `formatNote` (KK)

```
БІРЫҢҒАЙ ФОРМАТ:
"question" — қазақ сөйлемі "_____" пропускімен.
"options" — 4 қазақша нұсқа.
"explanation" — қазақ тілінде, барлық 4 нұсқаны талда.
```

### `rulesCatalog`

```
СПИСОК ДОПУСТИМЫХ ТЕМ (rule_id из /basics — только из этого списка):
- rule_01 — Гармония гласных (сингармонизм)
- rule_02 — Ассимиляция согласных
- rule_03 — Множественное число существительных
- rule_04 — Падежные окончания
- rule_05 — Притяжательные окончания
- rule_06 — Степени сравнения прилагательных
- rule_07 — Настоящее (актуальное) время
- rule_08 — Прошедшее время
- rule_09 — Будущее время
- rule_10 — Отрицательная форма глагола
- rule_11 — Порядок слов SOV
- rule_12 — Вопросительные предложения
- rule_13 — Послелоги
- rule_14 — Числительные 1–20 и порядковые
- rule_15 — Личные местоимения
- rule_16 — Словообразовательные суффиксы
- rule_17 — Наклонения глагола
- rule_18 — Специфические звуки
- rule_19 — Заглавные буквы
- rule_20 — Причастные формы
- rule_22 — Настояще-будущее -ады/-еді
- rule_23 — Указательные местоимения
- rule_24 — Отрицание именных предложений (емес)
- rule_25 — Аффиксы сказуемости

В каждом упражнении ОБЯЗАТЕЛЬНО заполняй поле "rule_id" одним из значений выше. Темы вне списка (каузатив, страдательный залог и т.п.) ЗАПРЕЩЕНЫ — иначе ученик не сможет открыть правило по ошибке.
```

## User-сообщение

```
RU: Сделай 5 упражнений по теме «{lessonTitle || topic}» (категория: {topic}, уровень {level}, режим: {difficultyTag}{scorePct ? ', avg_score=' + scorePct + '%' : ''}).
KK: {lessonTitle || topic} тақырыбынан {level} деңгейіне сай 5 жаттығу жаса (категория: {topic}, режим: {difficultyTag}{scorePct ? ', avg_score=' + scorePct + '%' : ''}).
```

## Как переиспользовать

1. **Заменить язык/предмет** — «учитель казахского языка» → «учитель X». Список rule_id заменить на свой каталог тем (1 rule = 1 страница в `/learn/basics`).
2. **Заменить формат вопроса** — текущий заточен под `Mother tongue + target language` (двуязычный fill-in-blank). Для одноязычных предметов (физика, история) — упростить.
3. **CEFR-шкала** работает только для языков. Для других предметов — заменить на свою шкалу (классы 1–11 для школы, beginner/intermediate/advanced для курсов).
4. **`rule_id` обязательность** — ключевая фича: позволяет UI открывать «правило по ошибке». Если каталога правил нет — убрать поле или сделать `rule_id: null`.
5. **Адаптивная сложность** через `avg_score` → стандартный паттерн, переиспользуется как есть.
